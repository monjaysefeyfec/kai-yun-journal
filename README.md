# kai-yun-journal

## 项目简介

`kai-yun-journal` 是一个用于归档和发布独立 HTML 页面的仓库。这里存放了不同时期、不同主题的静态页面，它们以独立的文件形式存在，不依赖于任何特定的后端服务或数据库。

每个 HTML 页面均可通过浏览器直接打开查看，适合作为个人或小型项目的页面存档、资料整理或内容展示。

## 目录说明

```
kai-yun-journal/
├── index.html          # 仓库主页（可选）
├── pages/              # 存放主要 HTML 页面
│   ├── example-1.html
│   ├── example-2.html
│   └── ...
├── assets/             # 页面使用的资源文件
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：核心目录，每个 `.html` 文件代表一个独立页面。
- `assets/`：存放页面所需的样式、脚本、图片等静态资源。

## 页面归档说明

- 所有页面均为纯前端 HTML，无需服务器环境即可运行。
- 页面内容保持原始结构，可能包含排版、样式或脚本，以文件创建时的状态为准。
- 归档页面可能不会跟随外部链接或引用资源的变化而更新，部分资源可能已不可用。
- 页面之间的链接或跳转仅在本仓库内有效，不指向外部站点。

## 维护说明

- 本仓库仅用于归档和发布，不提供内容更新通知或版本管理外的额外服务。
- 如需新增页面，请将 HTML 文件及关联资源放入对应目录，并确保相对路径正确。
- 不建议对已归档页面进行大规模修改，以保留原始记录。
- 欢迎通过 Pull Request 提交页面，但请注意保持内容独立性和仓库简洁性。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 使用浏览器打开 `pages/` 目录下的任意 `.html` 文件即可查看。
3. 若通过 GitHub Pages 部署，可直接访问对应 URL 浏览页面（需自行配置）。
